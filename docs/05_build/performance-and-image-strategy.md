# Performance & Image Strategy

*Owner: Frontend Architect*

The "GreatWood" brand proposition relies heavily on high-fidelity, emotional photography. However, a slow site destroys trust immediately. We must balance premium visual quality with aggressive performance optimization.

## 1. Image Optimization Pipeline
*   **Format:** WebP and AVIF formats strictly enforced via Next.js `next/image` component.
*   **Delivery:** Vercel Edge Network image optimization.
*   **Sizing:** Strict use of the `sizes` attribute in `next/image` to ensure mobile devices do not download 2880px desktop heroes.
    *   *Example:* `sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"`
*   **Quality:** Default to `quality={85}` for a balance of retina-sharpness and file size, dropping to `80` for secondary Lookbook imagery.

## 2. Preventing Cumulative Layout Shift (CLS)
*   **Aspect Ratio Locking:** Every image container must have a fixed aspect ratio (e.g., `aspect-video` or `aspect-[4/5]`) defined in CSS *before* the image loads.
*   **Placeholders:** Use blurry baseline Data URIs (LQIP - Low Quality Image Placeholders) generated by Sanity at build time, passed into the `blurDataURL` prop of `next/image`.

## 3. The 5-Shot Minimum Loading Strategy (PDP)
1.  **Establishing Hero:** `priority={true}`. Bypasses lazy loading. Initiated immediately in the document `<head>` via preload hints.
2.  **Contextual Lifestyle:** Lazy loaded (`loading="lazy"`).
3.  **Macro Material:** Lazy loaded.
4.  **Structural Integrity:** Lazy loaded.
5.  **Scale / Human Reference:** Lazy loaded.

## 4. Third-Party Scripts & Bloat
*   Analytics, customer support chat widgets, and marketing pixels must be loaded strictly *after* the page becomes interactive (using Next.js `next/script` with `strategy="lazyOnload"` or `afterInteractive`).
*   The primary font files (e.g., Monument Extended, Inter) must be preloaded locally or via `next/font` to eliminate layout jank and FOIT (Flash of Invisible Text).
